Help & Guides

How to create a GitHub PAT
To trigger updates manually, a Personal Access Token (PAT) is required to identify you.
  1. Go to your GitHub account settings and select Developer Settings.
  2. Click Personal Access Tokens → Tokens (classic).
  3. Click Generate new token (classic).
  4. Give the token a description, and select the repo checkbox (or workflow checkbox).
  5. Click Generate token at the bottom.
  6. Copy your token and paste it into the Settings form on this page.
⚠️ Note: For your security, this token is stored purely client-side in your browser's LocalStorage. It is never uploaded to any external server.
Troubleshooting
Common issues:
  • 401 Unauthorized: Double check your GitHub token. Ensure it is correct and not expired.
  • 404 Not Found: Verify the Repository Owner and Repository Name fields are correct and match your GitHub URL path exactly.
  • Permission Error: Make sure the GitHub token has the repo or workflow permission scopes activated.
